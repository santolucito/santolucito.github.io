Assignment 3: Mastermind

In this assignment, we're going to leave our friendly snakes aside and program a version of the game Mastermind. This is how this variation works:

  1. There are two players
  2. Player 1 picks 4 random distinct digits between 1 and 7, and remembers their order (call it the secret)
  3. Player 2 continually guesses 4-distinct-digit strings.
  4. Player 1 responds with 2 numbers: the number of digits in Player 2's guess that appear in the secret, and the number of numbers that are in the right place within the secret. For example, if the secret is "1234" and the guess is "1467", the response will be "2,1" because 1 and 4 are in both strings, but only 1 is in the right place.

Making it tougher

What's described above is almost the same as regular Mastermind. But since we are creative Yalies, we're going to add some tricky features to make the game more fun and challenging:

  1. Player 1 can lie with 33% probability. If Player 1 lies, only one number can be changed (i.e. from "1,0" to "1,1", but not to "2,1". Player 1 may not lie twice in a row. Lastly, lies must be valid responses, that is, both numbers must be between 0 and 4 and the second one must be less than or equal to the first. "1,2" and "5,3" are both impossible and cannot be lies.

  2. Player 2 must always make a guess that's greater than all of his or her previous guesses, with 3 exceptions. If all three of these exceptions are used up, and mySecret is less than the player's last guess, the player loses.

    For example, if the secret is "4356":

    1. "4123" (valid)
    2. "1567" (valid, 2 exceptions left)
    3. "2345" (valid, 1 exception left; exceptions are counted from the greatest guess thus far, so even though this guess is larger than the previous guess, it still uses up an exception)
    4. "3251" (valid, 0 exceptions left)
    5. "4125" (valid, it's greater than 4123)
    6. "4124" (invalid, it's less than 4125)
    7. "5213" (valid, but player loses because 5203 > 4356)

Part 1: My Little Secret (4 points)

Implement the method makeMySecret. The method should set the static variable mySecret to a random 4-character string, where each "letter" is a digit between 1 and 7. No digit may appear twice, and only the 7 digits from 1 - 7, inclusive, are allowed. You'll notice that when DEBUG is true, the template code prints the secret to the console. Please do not change this line. This will be helpful to you when testing, as well as to the TAs when grading.

Valid examples: "1234", "7653"

Invalid examples: "1123", "12345", "1a34", "123", "0123", "8651"

Part 2: Check Validity (2 points)

Implement the isGuessValid method, which takes as an input the string the user has input. Check if the input string is a valid input, that is, a 4-letter string where all the characters are distinct digits between 1 and 7 (inclusive)—the guess must meet the same criteria as mySecret. If the guess is invalid, print the message "Input must be a 4-digit number with digits between 1 and 7." and return false. Otherwise, print nothing and return true.

Part 3: Process the Guess (7 points)

Implement the method isGameOver, which takes as an input the string the user has input, and returns a boolean.

  1. Call isGuessValid with the input. If the return value is false, then return false without doing anything. The game should then just proceed to the next turn.

  2. Compare the input against mySecret. Print out the guess and the two number response, separated by a comma (and no spaces). The first number is the number of digits the input and mySecret have in common. The second number is the number of digits that are in common and in the correct position. Note that the second number is always less than or equal to the first number.

    For example, if the secret is "6573"

    1. Guess: "1234". Response: Guess: 1234; Result: 1,0 (3 appears in both strings, but no digits are in the correct place)
    2. Guess: "6132". Response: Guess: 6132; Result: 2,1 (6 and 3 both match, but only 6 is in the right place)
    3. Guess: "5673". Response: Guess: 5673; Result: 4,2

  3. If the user gets the input correct, print "You won!" instead of "4,4", and return true. Otherwise, return false.

Part 4: Extras (5 points)

Implement the extra features.

  1. Keep track of the number of exceptions remaining and use an exception up if the current guess is lower than the highest guess that player 2 has guessed thus far. This check should happen before the comparison in Part 3 (2). If the guess is lower than a previous guess, print "Your guess was lower than allowed. You have 2 exceptions remaining." (replace 2 with the actual number remaining). You should say "1 exceptions remaining." even though that's not grammatical. If the user makes an invalid guess and has no exceptions remaining, then you should still print out "Your guess was lower than allowed. You have 0 exceptions remaining.". However, if the user has no exceptions remaining and has guessed a number higher than the correct one, you should also print "You're out of exceptions and you've guessed too high! The secret was 1234." and return true, indicating that the game is over. (Of course, replace 1234 with the value of mySecret).
  2. When responding in isGameOver, lie with a 33% probability if you didn't lie the turn before. The lie should be a valid, but incorrect, response that changes only one of the two numbers. The number it changes should be chosen randomly, as well as the new value. (Make sure that the value that is randomly generated by the computer is, in fact, a lie. In other words, you should regenerate the random number if the random number generated by the compuer is accidentally correct and not acutally a lie.) If your guess is a lie and the global variable DEBUG is true, your program should additionally print "Lie" to the screen. Note that the computer can still lie, even if the user has won! But the computer should never lie with "4,4" or "4,3" because, well, then it's just obvious that it's a lie.

Messages

For convenience, here are all the messages you should make sure your program outputs exactly at the right times. Whenever x's are used below, please replace with the appropriate value.

  1. "Secret: xxxx", when DEBUG is on, right after the program generates mySecret (this is given to you in the template)
  2. "Input must be a 4-digit number with digits between 1 and 7.", when there is an invalid input
  3. Guess: xxxx; Result: "x,x" when a proper input is given
  4. "Your guess was lower than allowed. You have x exceptions remaining.", when the guess is lower than any previously guessed number.
  5. "You're out of exceptions and you've guessed too high! The secret was xxxx." when the above applies, but the user has no exceptions left. Note that the game ends after this message is displayed.
  6. "Lie", when DEBUG is true and the computer responds with a lie.
  7. "You won!", when the correct number is guessed. The game ends after this message.
